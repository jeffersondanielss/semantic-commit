Semantic Commit

Tipo e descrição de commits, que destacam exatamente o que foi mudado no código.

Problem

Trabalhando em equipe, cada um escrevendo a sua maneira, commits tendem á viram uma bagunça.

➜ git log --pretty=oneline
29d5g2 (tag: 0.3.0) add new awesome feature
6das1n  fix
n3282e (tag: 0.2.3) bump version
b2h4vs  resolve conflict
43h43b  merge with branch BugFixTitle

Solution

Com um padrão de commit estabelecido, podemos ter commits uniformes.

➜ semantic-commit
? Choose the type of the commit: (Use arrow keys)
  chore
  docs
  feat
> fix
  refactor
  style
  test

Pattern

Com isso é muito mais localizar quando e como as mudanças no código foram feitas.

➜ git log --pretty=oneline
29d5g2 (tag: 0.2.3) fix: title of page
29d5g2 (tag: 0.2.3) fix: title of page

Install

Para usar basta instalar o pacote com seu gerenciador de dependências favorito.

# using yarn run
➜ yarn global add semantic-commit
# using npm run
➜ npm i -g semantic-commit
